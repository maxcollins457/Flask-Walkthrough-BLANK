Using Jinja

Jinja is a powerful and widely-used template engine for Python. It is primarily used in web development to generate dynamic HTML, XML, or other markup formats. Jinja allows developers to embed logic, variables, and control structures within templates, which are then rendered to produce the final output displayed in a web browser.

Accessing variables

Any variables that are passed to the html file can be accesses using Jinja

Welcome, {{ user_info.username }}

Conditional Statements

We can use conditionals to only show elements based on a condition

{% if user_info.full_name %}

Full Name: {{ user_info.full_name }}

{% endif %}

Loops

We can loop through variables, much as we would in python

Filters

Jinja filters can be used to change variables e.g. reverse

Bio: {{ user_info.bio|reverse }}

Filters with Arguments

Some filters take arguments, such as replace

Email: {{ user_info.email|replace('example', 'gmail') }}

Macros (Reusable Blocks)

Macros provide resulable blocks of jinja, these can even be written in a seperate html file and imported

{% macro display_website_link(url) %} {{ url }} {% endmacro %}

Website: {{ display_website_link(user_info.website) }}
